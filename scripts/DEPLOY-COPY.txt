KOON — what to copy on the server (2 steps)
==============================================

1) Laravel root (same level as public_html)
   Either merge the bundle into HOME so artisan lives next to public_html (koon.edu.sa),
   OR upload the whole "koon-hosting" folder as ~/koon-hosting (zip layout).
   You must see: artisan, vendor/, app/, public/, ...

2) public_html (web root)
   Copy EVERYTHING that is INSIDE koon-hosting/public/
   into public_html/ (overwrite). Do not upload the whole koon-hosting zip into
   public_html — only the inner "public" files.

public_html/index.php: $laravelFolder = '' when Laravel is in HOME;
  $laravelFolder = 'koon-hosting' when Laravel is in ~/koon-hosting.
secrets/sftp.env: SFTP_APP_DIR empty (HOME) or SFTP_APP_DIR=koon-hosting.

Then: .env in the Laravel root, php artisan key:generate, migrate, storage:link.
See HOSTING.txt for details.

3) Automated SFTP (after publish-hosting.ps1)
   Credentials: copy secrets/sftp.env.example -> secrets/sftp.env (gitignored).
   Or legacy ~/Downloads/sftp.txt (host:/user:/pass:).
   From repo root:
     pip install paramiko
     python scripts/sftp_deploy.py              # incremental vs publish/last
     python scripts/sftp_deploy.py --full         # upload everything
     python scripts/sftp_deploy.py --dry-run
   Or: .\scripts.ps1 -Action sftp-sync | sftp-sync-full | publish-sftp
   Incremental compares to publish/last/koon-hosting (updated after each success).
